Liste des classes

Liste des classes, structures, unions et interfaces avec une brève description :
CCentralWidget
CCuteCom
CDataConteneur pour les données
CDeviceClasse de device
CDockWidgetDeviceMonitor
CDockWidgetListDataService
CDockWidgetPlugins
CDockWidgetServicesDockwidget qui contient la liste des services, initialise la liste des services, lance les services et gère les connections entre services et plugins
CDockWidgetServicesStackedWidgetDockWidget qui contient les widgets de tout les services
CListDataReprésente une liste de donnée. Cette classe possède les accesseurs pour manipuler les données
CListDataPluginCette classe reprèsente la liste des données d'un plugin. Elle stocke plusieurs liste de donnée, ici les paramètres de connections et les données
CListDataServiceCette classe reprèsente la liste des données d'un service. Un service à accée à toute les informations de tout les plugins. C'est donc une liste de liste de données de plugin
CMainWindowWidget principale de l'application qui contient les autres widgets
CPluginAbstractClasse abstraite qui contient la déclaration des méthodes génériques aux plugins
CPluginAbstractObjectClasse abstraite qui contient la déclaration des méthodes génériques aux plugins pour les slots et signaux
CPluginAbstractSerialPortClasse abstraite des plugins port série
CPluginBluetoothClasse du plugin bluetooth. Permet de reconnaitre les périphériques bluetooth, et d'envoyer des fichiers sur un périphériques. L'association des appareil n'est pas géré pour l'instant
CPluginBluetoothThreadCe thread surveille et détecte les périphériques bluetooth
CPluginBluetoothThreadSendFileCe thread est lancé lors de l'envoi de fichiers
CPluginCANGineClasse pour gérer le cangine. Pour détecter la déconnection du périphérique il faut surveiller avec un timer. Le parsage du message est assez lourd, peut être faudrait t'il améliorer
CPluginCorsairCette classe est similaire au plugin d-note de la tablette tactile. Il faudrait faire un plugin plus générique avec libusb
CPluginDNoteX20DiskCette classe est similaire au plugin corsaire. Il faudrait faire un plugin plus générique avec libusb
CPluginGPRSCette classe gère le gprs. La difficulté est la gestion du protocole de communication
CPluginGPSCette classe gère le gps
CPluginNetworkCette classe permet de récupérer les adresses réseaux
CPluginSerialPortCette classe se connecte en serial port si les périphériques connues ne sont pas détectés. En faite elle n'est pas très utile, sauf pour le programmeur. Attention, une seul instance de chaque plugin est utilisable, donc si deux périphérique sont reconnu, un seul s'associera avec le plugin
CServiceAbstractCette classe reprèsente la classe abstraite des services
CServiceAbstractObjectClasse abstraite qui contient la déclaration des signaux et slots des services
CServiceCopyCorsairToDataService de copie des fichiers de la clé usb corsaire. Il faudrait peut être repasser avec libusb pour être plus générique
CServiceCopyCorsairToDataThreadLe thread copie les fichiers, il faudrait améliorer en ne copiant les fichiers que si c'est necessaire avec peut être un QFilsSystemWatcher
CServiceCopyDNoteDiskToDataService de copie des fichiers de la tablette tactile vers le répertoire de travail
CServiceCopyDNoteDiskToDataThreadLe thread copie les fichiers périodiquement, il faudrait peut être le faire que quand c'est nécessaire. Voir QFileSystemWatcher
CServiceFliteService de voix synthétique. La librairie utilisé est flite-1.4-release, les versions antérieur ne marche pas
CServiceFliteThreadThread qui lit les informations à lire à la suite
CServiceSaveFileService d'enregistrement des données dans un fichier text, il faut revoir des fois c'est pas très propre dans le fichier (quand il y a plusieur info à la suite). On peut aussi faire le même service mais en enregistrant un fichier xml
CServiceSaveFileHtmlPermet de générer un fichier html, si on active le service ftp envoi directement le fichier html sur un serveur. Mais des fois à cause des rafraichissement intempestif et des copie sur le serveur, le fichier html s'affiche mal dans le navigateur, c'est pour ca qu'il y a aussi la possibilité de passer par le service de base de donnée
CServiceSaveFileHtmlThreadThread enregistre les informations cochées dans le treeWidget dans un fichiers, et génére un fichier html
CServiceSaveFileThreadThread enregistre les informations cochées dans le treeWidget dans un fichier
CServiceSendBluetoothLes informations sont transmis au plugin bluetooth par l'intermediaire de la liste de donnée, c'est lui qui envoi les fichiers. On aurait pu le faire dans le service, mais si un autre service à besoin de cette fonctionnalité il faut la réécrire
CServiceSendDataBase
CServiceSendFtp
CServiceSendFtpGprs
CServiceSendFtpGprsThread
CThreadDeviceMonitorCette classe gère le thread qui surveille la connection des périphériques ou la déconnection
Ui::CuteCommDlg
